Problem Solving

05 December 2021

Since we started JavaScript, problem-solving techniques have been present daily in my studies. I will tell you about the Super FizzBuzz problem done this week. The narration of the exercise was a long description of it - point one of why I got stuck. Well, my friends, I learned that I tend to overthink if there is too much information. We had to print “Fizz” instead of multiples of 3, “Buzz” instead of multiples of 5, and “FizzBuzz” for multiples of 15.

I had a good start using the conditionals, but suddenly I decided to also add loops to consider only numbers from 1 to 100. Please note that the exercise expected us to write two different functions, but I ended up trying to include the full problem in just one. It was just not working. I tried console log the variables, but there were so many, that I only got more confused.

That’s why I decided that the whole thing was just not making sense. Googling the original problem was my first attempt to have different perspectives. It worked just fine! The first thinking about the conditionals was perfect. Then, at this moment, I realised that my interpretation of the problem seemed to be too superficial. I got back to the heading and decided to write down a summary on a piece of paper. After that, I read it all again and included some points, or deleted any extra information.

Nice! At the end of this process, I felt fresh to start again as new ideas came up. I could finally see how I had to break down the problem. The first function would be the normal FizzBuzz, and the loop would happen only on the second function. Console log finally worked fine on the first problem - ! Then, I started the second part of the loop. It was way simpler to write down:

  1. One variable with an empty array
  2. The loop would now happen as per the numbers given by the user (no range from 1-100)
  3. Another variable “result name” to allocate the result of the number when the Fizzbuzz function was used - being the same number or “Fizz”, or ‘Buzz, or “FizzBuzz”
  4. Add this “result name” to the new array

And… It was still not working . I had to include another break here because the lines looked all like the same here. Frustration was the definition of my spirit.

It was just not working, so I asked for help! My partner was the first one to put his hands up. It was indeed great to confirm that so much of the coding was right. But it seemed odd that the numbers were not getting pulled individually from the array to be checked. So here I went again. Add a new variable to collect the number from the array. This number would then get compared to the fizzbuzz function and allocated on the resultName. And this is finally how I won this problem!

As you can see, the problem took a long process to get completed (and time). But I counted on some great techniques to help me to keep calm and investigate new concepts. These were:

  • Pseudocode: picture the situation and write down the steps to be followed
  • Trying something
  • Google: never old - always have something to add to the cause
  • Console.logging: checking what the computer is doing
  • Improving your process with reflection
  • Asking your peers for help
